# Technical SEO Audit

## Overview
A technical SEO audit evaluates whether a website can be properly crawled,
indexed, and understood by search engines.

Many search visibility problems originate from technical barriers rather than
content quality. A technical audit identifies these barriers and provides a
prioritized list of fixes.

## Best Fit
This service is appropriate when a site experiences ranking loss,
indexing problems, or large fluctuations in organic traffic.

It is also recommended before major SEO initiatives to ensure that the
technical foundation supports future improvements.

## Deliverables
Typical deliverables include:
- crawl diagnostics
- indexing analysis
- redirect and canonical review
- site architecture evaluation
- prioritized technical recommendations

## Inputs Required
Access to Google Search Console
Access to analytics platforms
Crawl access to the website
Server response data when available

## Typical Findings
Common issues discovered during audits include:
- broken redirects
- orphan pages
- crawl traps
- duplicate content signals
- incorrect canonical tags

## Exclusions
The audit identifies issues but does not automatically include
implementation of fixes unless separately scoped.

## Prerequisites
The website must allow crawl access and provide basic analytics data.

## Success Criteria
Success is measured by improved indexing stability,
reduced crawl errors, and clearer technical architecture.

## Recommended Pairings
Often paired with indexing remediation or on‑page optimization.

## Notes
Technical audits frequently serve as the diagnostic starting point
for larger SEO engagements.
